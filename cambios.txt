1. INCONSISTENCIA CRÍTICA EN ENUMS DE ESTADOS (OrderStatus)
--------------------------------------------------------------------------------
ARCHIVO: /workspace/frontend/src/types/index.ts
PROBLEMA: Los valores del enum OrderStatus estaban en minúsculas ('pendiente', 
          'asignado', etc.) pero todo el código restante usaba mayúsculas.
IMPACTO: Error al iniciar sesión y comparar estados en toda la aplicación.

CORRECCIÓN:
  ANTES:
    export enum OrderStatus {
      PENDIENTE = 'pendiente',
      ASIGNADO = 'asignado',
      EN_PREPARACION = 'en_preparacion',
      ...
    }
  
  DESPUÉS:
    export enum OrderStatus {
      PENDIENTE = 'PENDIENTE',
      ASIGNADO = 'ASIGNADO',
      EN_PREPARACION = 'EN_PREPARACION',
      ...
    }

--------------------------------------------------------------------------------

3. INCONSISTENCIA EN REGISTRO DE REPARTIDORES (vehicle_type)
--------------------------------------------------------------------------------
ARCHIVO: /workspace/frontend/src/app/(auth)/register/page.tsx
PROBLEMA: El formulario de registro usaba valores en inglés:
          ['bicycle', 'motorcycle', 'car']
          mientras que el sistema espera valores en español y en mayúsculas.

CORRECCIÓN:
  ANTES:
    vehicle_type: z.enum(['bicycle', 'motorcycle', 'car'])
    const vehicleOptions = [
      { value: 'bicycle', label: 'Bicicleta' },
      { value: 'motorcycle', label: 'Motocicleta' },
      { value: 'car', label: 'Automóvil' },
    ];
  
  DESPUÉS:
    vehicle_type: z.enum(['BICICLETA', 'MOTO', 'AUTO'])
    const vehicleOptions = [
      { value: 'BICICLETA', label: '🚴 Bicicleta' },
      { value: 'MOTO', label: '🏍️ Motocicleta' },
      { value: 'AUTO', label: '🚗 Automóvil' },
    ];

--------------------------------------------------------------------------------
